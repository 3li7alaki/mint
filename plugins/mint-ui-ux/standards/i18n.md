# Internationalization (i18n) Standards

Standards for translatable UI code. Enforced in design review.

## Core Principle

**No hardcoded user-facing strings.** Every piece of text a user sees must be translatable.

## What Must Be Translated

### Always Translate

- Page titles and headings
- Button labels
- Form labels and placeholders
- Error messages
- Success messages
- Empty state messages
- Navigation items
- Tooltips
- Modal titles and content
- Table headers
- Dropdown options (user-facing)

### Usually Translate

- Alt text for images (unless decorative)
- ARIA labels
- Meta descriptions
- Open Graph titles

### Don't Translate

- Code/technical identifiers
- URLs and paths
- Console logs (dev only)
- API keys and tokens
- File names (usually)

## Review Severity

| Violation | Severity |
|-----------|----------|
| Hardcoded button label | **BLOCKING** |
| Hardcoded error message | **BLOCKING** |
| Hardcoded heading/title | **BLOCKING** |
| Hardcoded form label | **BLOCKING** |
| Hardcoded placeholder | **WARNING** |
| Hardcoded tooltip | **WARNING** |
| Hardcoded alt text | **WARNING** |
| Inline language conditionals | **BLOCKING** |
| Fallback strings with `\|\|` or `??` | **BLOCKING** |

## Anti-Patterns

### ❌ Inline Language Conditionals

```tsx
// BLOCKING - never do this
{locale === 'ar' ? 'مرحبا' : 'Hello'}
{lang === 'en' ? 'Submit' : 'إرسال'}
```

Use proper i18n system instead.

### ❌ Fallback Strings

```tsx
// BLOCKING - translation might be missing, fix the translation file
const label = content.submit || 'Submit';
const title = t('title') ?? 'Default Title';
```

If a translation doesn't exist, add it to the translation file.

### ❌ String Concatenation for Sentences

```tsx
// BAD - word order differs between languages
`You have ${count} new messages`

// GOOD - use interpolation
t('messages.count', { count })
// en: "You have {count} new messages"
// ar: "لديك {count} رسائل جديدة"
```

### ❌ Partial Translations

```tsx
// BAD - mixing translated and hardcoded
<span>{t('price')}: $99</span>

// GOOD - full translation with formatting
<span>{t('price_value', { price: formatCurrency(99) })}</span>
```

## Framework Patterns

### React with Intlayer

```tsx
import { useIntlayer, str } from '@/lib/i18n';

function MyComponent() {
  const content = useIntlayer('namespace');

  return (
    <>
      {/* JSX children - works directly */}
      <h1>{content.title}</h1>

      {/* String props - use str() */}
      <Input placeholder={str(content.placeholder)} />
      <button aria-label={str(content.ariaLabel)}>
        {content.buttonText}
      </button>
    </>
  );
}
```

### Vue with vue-i18n

```vue
<template>
  <h1>{{ $t('title') }}</h1>
  <input :placeholder="$t('placeholder')" />
  <button :aria-label="$t('ariaLabel')">
    {{ $t('submit') }}
  </button>
</template>
```

### Vue with Nuxt i18n

```vue
<template>
  <h1>{{ $t('page.title') }}</h1>
</template>

<script setup>
const { t } = useI18n();
const message = t('welcome');
</script>
```

## Translation File Organization

### Recommended Structure

```
i18n/
├── en/
│   ├── common.json      # Shared strings
│   ├── auth.json        # Auth pages
│   ├── dashboard.json   # Dashboard
│   └── errors.json      # Error messages
└── ar/
    ├── common.json
    ├── auth.json
    ├── dashboard.json
    └── errors.json
```

### Key Naming

```json
{
  "auth": {
    "login": {
      "title": "Sign In",
      "submit": "Sign In",
      "forgotPassword": "Forgot password?",
      "errors": {
        "invalidCredentials": "Invalid email or password",
        "accountLocked": "Account is locked"
      }
    }
  }
}
```

## Numbers, Dates, Currency

Always use Intl formatters:

```tsx
// Numbers
new Intl.NumberFormat(locale).format(1234.56)
// en: "1,234.56"
// ar: "١٬٢٣٤٫٥٦"

// Currency
new Intl.NumberFormat(locale, {
  style: 'currency',
  currency: 'USD'
}).format(99.99)

// Dates
new Intl.DateTimeFormat(locale, {
  dateStyle: 'long'
}).format(date)

// Relative time
new Intl.RelativeTimeFormat(locale).format(-1, 'day')
// en: "1 day ago"
// ar: "منذ يوم واحد"
```

## Pluralization

Different languages have different plural rules:

```json
// English - 2 forms (one, other)
{
  "items": "{count, plural, one {# item} other {# items}}"
}

// Arabic - 6 forms (zero, one, two, few, many, other)
{
  "items": "{count, plural, zero {لا عناصر} one {عنصر واحد} two {عنصران} few {# عناصر} many {# عنصرًا} other {# عنصر}}"
}
```

Use ICU MessageFormat or your framework's plural support.

## Testing Translations

1. **Pseudo-localization**: Replace strings with accented versions to spot hardcoded text
2. **Length testing**: Some languages are 30-50% longer than English
3. **RTL testing**: Arabic/Hebrew layouts
4. **Missing key detection**: CI check for untranslated keys

## Adding New Strings

1. Add to translation file first
2. Use the key in code
3. Never commit code with hardcoded strings
4. Run translation check before commit

```bash
# Framework-specific translation checks
pnpm translation:check     # Intlayer
pnpm i18n:check           # vue-i18n
```
